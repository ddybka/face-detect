Так. Ты сейчас бегаешь по кругу между двумя ошибками, и каждый «фикс» одного пакета ломает другой, потому что pip 21 без конца перетягивает версии. Это не твоя криворукость — это реально гнилая комбинация на Python 3.9. Хватит латать. Снесём весь TF-стек подчистую и поставим одной атомарной командой с жёсткими пинами всего сразу, чтобы pip физически не мог подтянуть свежак.

В venv, по шагам:

1. Снеси весь keras/tensorflow-зоопарк целиком:

bash
pip uninstall -y tensorflow tensorflow-intel tensorflow-estimator tensorboard tensorboard-data-server keras tf-keras tf_keras ml-dtypes
2. Поставь всё четырьмя жёсткими пинами в ОДНОЙ команде (так pip решает зависимости совместно и не может поставить чужие версии):

bash
set PYTHONIOENCODING=utf-8
pip install --no-compile "tensorflow==2.15.0" "keras==2.15.0" "tensorboard==2.15.2" "ml-dtypes==0.2.0" "numpy<2"
3. Проверь — обе версии обязаны быть 2.15.0:

bash
python -c "import tensorflow as tf; print('tf', tf.__version__); import keras; print('keras', keras.__version__); from tensorflow.keras.models import Sequential; print('OK')"
Нужно: tf 2.15.0, keras 2.15.0, OK.

4. Убери из скрипта строку про legacy keras — с чистым keras 2.15 она не нужна и может путать. Первые строки 3-face-recognition.py должны быть просто:

python
import os
import cv2
from deepface import DeepFace
(строку os.environ["TF_USE_LEGACY_KERAS"] удали).

5. Запусти:

bash
python .\3-face-recognition.py
Почему теперь должно встать намертво: keras 3 не появится, потому что мы его не ставим и не даём зависимостям его подтянуть (keras==2.15.0 зафиксирован явно в той же команде). tf-keras удалён — он был источником перетягивания TF 2.20. Это согласованный комплект, который не дрейфует.

Если шаг 3 опять покажет не те версии или ошибку — пришли его вывод дословно, и я скажу точную причину. Но сначала пришли именно вывод шага 3, не шага 5 — корень всегда там.
